# 出题流程（/quiz）

## 第一步：确认出题范围

用户必须明确到 **课程 + 章节** 级别，才能进入出题流程。

- 若用户已指定章节（如 `/quiz 第三章`）→ 直接进入第二步
- 若未指定或只说"出题"：

  ```bash
  python {SKILL_DIR}/commands/quiz.py --list
  ```

  将输出的章节列表展示给用户，**询问选择哪个章节**，等待回复后再继续。

---

## 第二步：加载知识点上下文

```bash
# 按章节加载
python {SKILL_DIR}/commands/quiz.py --chapter "{章节名}"

# 若用户指定了具体知识点
python {SKILL_DIR}/commands/quiz.py --topic "{知识点名}"

# 若用户说"考我薄弱点"
python {SKILL_DIR}/commands/quiz.py --weak
```

输出以 `QUIZ_CONTEXT:` 开头，包含各知识点的定义、示例、应用和题型建议。

---

## 第三步：提炼核心概念

从 `QUIZ_CONTEXT` 中选出 **3-5 个** 本次出题的核心概念（优先选 `★ 薄弱点` 标记的，其次按重要程度）。

---

## 第四步：搜索真题 / 模拟题

对每个核心概念执行联网搜索：

```
搜索词示例："{概念名称} 例题"、"{概念名称} 练习题"、"{课程名} {概念} 真题"
```

**判断标准**：
- 搜索结果中有与该知识点**高度吻合**的题目（题干、考点明确对应）→ **直接引用**，注明来源
- 结果不相关、或只有泛泛介绍 → 进入第五步自主出题

---

## 第五步：自主出题

根据 `QUIZ_CONTEXT` 中的 **题型建议** 字段决定题型：

| 题型建议 | 出题规则 |
|----------|---------|
| 记忆型（选择/判断） | 围绕 `d` 字段（定义）出题；选择题设 4 个选项，错误选项要合理（不能一眼看出）；判断题给出判断依据 |
| 数学型（计算题） | 基于 `e` 字段中的示例改编，或自拟同类型数值；难度定为中等（不超过两步推导）；必须给出完整解题步骤 |

**每题输出格式**：

```text
【题目 N】{题干}

（选项 A/B/C/D — 仅选择题）

答案：{答案}
解析：{简要解析，指向知识点中的哪个概念}
来源：{自主出题 / 引用自 URL}
```

---

## 第六步：复杂度兜底

若概念满足以下任一条件，**不强行出题**，改为诚实告知：

- knowledge_base 中该知识点的 `e`（示例）字段为空或过于简略，无法支撑出题
- 概念涉及多步复杂推导（如多重积分变换、高阶微分方程证明），超出可靠出题范围

输出：

```text
⚠ "{概念名称}" 的题目超出当前出题能力范围（原因：{简要说明}）。
建议在以下途径搜索相关练习题：
  · 教材课后习题对应章节
  · 搜索 "{概念名称} 例题 解析"
```
