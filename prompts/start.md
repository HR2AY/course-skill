# 每日启动流程（/start）

## 第一步：获取上下文

```bash
python {SKILL_DIR}/commands/start.py
```

输出以 `START_CONTEXT:` 开头，包含昨日摘要、今日计划、本周主题、薄弱点。

---

## 第二步：打招呼

用自然、简短的语气开场，不超过两句话。根据时间段调整措辞：
- 早晨（5-10点）：早上好
- 上午/下午（10-18点）：你好 / 下午好
- 晚上（18点后）：晚上好

---

## 第三步：昨日回顾

若 `START_CONTEXT` 中有昨日记录：
- 有摘要（summary 字段）→ 直接用一句话转述
- 只有话题/完成数 → 用"昨天学了 {话题}，完成了 {N} 项任务"概括

若无昨日记录：
- 跳过，不强行编造

---

## 第四步：今日计划展示

将待完成任务以简洁列表展示，突出截止日期压力：

```text
📋 今日待完成
  · {任务内容}              [类型]
  · {任务内容}  ⚡ 后天截止  [类型]
  · {任务内容}              [类型]
```

若 calendar 中无今日计划（`暂无今日计划`），说明"根据本周规划，建议今天完成："，展示从学期规划推断出的任务。

---

## 第五步：推荐最佳启动任务

从待完成任务中选出**一个**最值得现在开始的任务，依据以下优先级判断：

| 优先级 | 条件 |
|--------|------|
| 最高 | 截止日期在 3 天内 |
| 高 | 任务类型是 `read`（阅读类，启动成本低） |
| 中 | 内容与当前薄弱点相关 |
| 低 | 任务类型是 `review` / `practice` |

**同等优先级时选内容最短的**（认知负担小，更容易开始）。

输出格式：

```text
🚀 建议从这里开始：
   {任务内容}
   理由：{一句话说明为什么选这个，例如"截止日期临近"/"内容较短适合热身"/"与你的薄弱点直接相关"}
```

---

## 注意事项

- 全程不超过 200 字，保持轻快
- 不要把所有字段原样列出，用自然语言转述
- 若今日已有部分任务完成，先肯定进度，再展示剩余
